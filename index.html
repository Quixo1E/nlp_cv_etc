Project Documentation

This document provides an overview of the functionalities, key methods, and data fields for each script in the project.

Link

General Requirements

  1. A conda environment based upon the libraries listed in environment.yml
  2. AirSim installed (from Colosseum), running on Unreal Engine (UE).
  3. ZED SDK installed.

Scripts Overview

follow_person_airsim.py

Connects the ZED camera on a laptop to an AirSim client. It employs a human detector network, captures the first person, reads the distance, and targets the drone to move toward that distance in meters.

Key Methods

  • connectToDrone(): Establishes connection with the drone.
  • findPerson(): Detects and captures the first person using ZED's human detection.
  • moveDrone(): Moves the drone towards the detected person.

Data Fields

  • client: AirSim client object.
  • person_location: Coordinates of the detected person.
  • drone_position: Current position of the drone.

airsim_toROS.py

Converts the AirSim API to ROS interpretable commands. It's designed to facilitate communication between AirSim and ROS for extended functionalities.

Key Methods

  • apiToRos(): Converts AirSim API commands to ROS messages.
  • rosSubscriber(): Subscribes to relevant ROS topics.
  • sendCommands(): Sends converted commands to the drone.

Data Fields

  • api_commands: Commands received from AirSim API.
  • ros_messages: ROS messages to be published.
  • current_state: Current state of the drone.
